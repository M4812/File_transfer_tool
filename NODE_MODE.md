# 节点运行说明

这个工具不是单独的“发送端程序”或“接收端程序”，而是部署在每台服务器上的“收发节点”。

## 运行方式

- 每台服务器都可以同时配置发送链路和接收链路。
- 接收链路负责监听本机端口，把远端文件落到本机指定目录。
- 发送链路负责监控本机目录，把目录中的正式文件发送到下一台服务器。

## 典型链路

### A -> B -> C

- A 服务器：
  - 配发送链路，监控 `A目录`，发送到 B。
- B 服务器：
  - 配接收链路，把 A 发来的文件接收到 `B接收目录`。
  - 再配发送链路，监控 `B接收目录`，发送到 C。
- C 服务器：
  - 配接收链路，负责最终落盘。

## 中转安全性

- 接收中的文件先写成 `.part` 临时文件。
- 只有接收完成后才改成正式文件名。
- 发送端默认忽略 `.part` 文件。

这样 B 作为中转节点时，不会把还没接收完整的半成品继续发给 C。
